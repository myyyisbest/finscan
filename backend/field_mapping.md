# finscan 数据源可行性验证报告 + 字段映射表

> 批次0 交付物 | 验证日期：2026-06-20 | 验证样本：贵州茅台 sh600519
> 验证环境：Python 3.12 + akshare 1.18.64 + baostock 0.8.9，本地运行

## 一、核心结论：数据源决策调整 ⭐

| 数据源 | 原计划（PRD） | 实测结果 | 最终决策 |
|--------|--------------|----------|----------|
| **Baostock** | 三表主数据源 | 🔴 **连接持续失败**（错误码 `10002007 网络接收错误`，3次重试+1次卡死超时，TCP端口可达但应用层断连） | **降级为备选源**，仅在新浪接口失效时兜底；不作为主依赖 |
| **AkShare-新浪三表** | 附注补充源 | 🟢 **远超预期**：资产负债表147列/利润表83列/现金流表71列，**完整官方财报科目**，回溯1998年，数值干净(float无脏数据) | ⭐ **升为三表主数据源** |
| **AkShare-东财财务指标** | 未规划 | 🟢 **完美覆盖** `fin_indicators` 全部指标（ROE/ROA/毛利率/净利率/偿债/周转/成长） | ⭐ 作为 `fin_indicators` 主源 |
| **AkShare-东财主营构成** | 附注源 | 🟢 行业/产品/地区三维度，含毛利率、收入占比 | 作为 `note_main_business` 源 |
| **AkShare-巨潮公告** | 公告源 | 🟡 接口可达但返回空（参数待调） | 后期（批次6）调参补全，非阻塞 |
| **应收账龄 / 前五大客户** | 附注源 | 🔴 AkShare **无现成接口** | 需从巨潮年报 PDF 解析（批次6实现），或规则降级 |
| **业绩预告/快报** | 风险源 | 🟡 接口存在，历史日期返回空 | 后期调参 |

### 决策影响
- ✅ **三表采集改用 AkShare 新浪接口**，字段比 Baostock 更全（PRD担心的"在建工程/长期待摊费用/商誉/其他应收款"全部齐全）
- ✅ `fin_indicators` 直接取东财预计算值，**口径由东财统一定义**（解决PRD评审点5"口径未定义"）
- ⚠️ 应收账龄(规则3.5)、前五大客户(规则5.5) 缺数据源 → 需 PDF 解析 或 规则降级为SKIP
- ⚠️ Baostock 保留依赖但加**健康检查+自动降级到新浪**的熔断逻辑

## 六、数据清洗规则（实测确认）

1. 新浪三表数值字段实测为干净 `float`，**无 "--" 脏数据**（茅台样本验证）
2. `报告日` 为字符串如 `20231231`，需转 `DATE`（`YYYYMMDD`→`YYYY-MM-DD`）
3. 空值统一存 `NULL`（PRD要求）
4. 单位已是**元**（实测货币资金=48786691397.55），无需单位换算
5. 三表按 `报告日` 倒序返回，含季报+年报，需按 `report_type` 分类（年末=Annual，季末日期判定Q1/H1/Q3）
